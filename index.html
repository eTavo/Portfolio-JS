Proyectos



integración con SQL Server

Realizar la extracción de las tablas de las bases de datos del cliente, en ocasiones con la autenticación de Windows y otras con credenciales de usuario y contraseña con permiso de lectura, parseando campos fechas, dependiendo la luz horaria del cliente, y generando en ocasiones CSV o Parquet con las librerías Pyarrow.Parquet, Pandas junto a los controladores necesarios para la instancia de SQLServer, para luego subir los archivos a los contenedores de Visionaris.

Integración con Jira

En este caso se tuvo que realizar la extracción de los datos de los issues registrados en el portal de Jira para poder tener los datos con los campos de asignación de usuario, quien lo genero, estado del mismo y para poder extraer el tiempo de trabajo, hubo que apuntarle a Tempo que se encuentra separado de Jira, teniendo en cuenta que no se crucen los datos de las incidencias. A su vez se tuvo que realizar un dataframe de los datos para descargarlos en formato parquet para luego integrarlos a Visionaris dentro de sus contenedores.

Migraciones entre distintos SO

Cuando un cliente cambia de servidor y al mismo tiempo de sistema operativo, pasándose de Windows Server a Linux o viceversa, se realiza un backup de mongodb para luego levantarlo en el nuevo servidor generando una nueva licencia del producto para que se mantenga la información al día. En estos casos se usa la consola de comandos Bash y CMD.

Integración con clientes Dividens

En esta oportunidad al tratarse de una Api, a través de las librerías de HttpClient, Resquests, Json y Pandas se realizo la conexión a la api teniendo en cuenta las distintas Apis para la extracción de diferentes datos, generando los dataframe y volcándolos en CSV.

Integración con plataforma Vtex

Para el caso de Vtex la plataforma cuanta con un sistema de token para las consultas a las diferentes Apis, caso similar al de Dividenz contando con librerías como HttpClient, Requests, Pandas y Json hubo que realizar la extracción, limpieza y transformación de datos ya que se encuentran en formato Json anidados para poder luego generar los dataframe para volcar la información que el cliente requería poder visualizar dentro de Visionaris.

Mejoras en la performance del Servicio

Al registrar diferentes reportes de clientes donde se veía afectada la performance del servicio que se brinda, decidí realizar un script en Batch en el cual se bajaban los motores de la app, se realizaba la limpieza de las carpetas temporales y se volvía a subir el servicio, con el cual soluciono un 80% del inconveniente en la mayoría de los clientes y manteniéndole la performance del servidor liberando espacio en el disco del servidor.
